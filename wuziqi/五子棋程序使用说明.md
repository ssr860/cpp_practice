# 五子棋小程序使用说明


## 1. 进入程序目录

打开 PowerShell，输入：

```powershell
cd D:\cpp\wuziqi
```

看到提示符类似下面这样，就说明位置正确：

```text
PS D:\cpp\wuziqi>
```

注意：`cd` 是进入文件夹，不是运行文件。不要输入 `cd train.py`。

## 2. 编译 C 程序

修改过 `wzq-s1-ASCII.c` 或训练完新权重后，都要重新编译：

```powershell
& "D:\Program Files\mingw64\bin\gcc.exe" .\wzq-s1-ASCII.c -o .\wzq-s1-ASCII.exe
```

如果没有报错，并且回到命令行提示符，说明编译成功。

## 3. 查看帮助

```powershell
.\wzq-s1-ASCII.exe --help
```

可用参数如下：

```text
--mode 1        人类执黑 vs AI执白
--mode 2        AI vs AI 自博弈
--mode 3        AI执黑 vs 人类执白
--log FILE      把每一步棋和棋盘保存到指定日志文件
--no-clear      终端不清屏，保留每一步显示
--max-moves N   最多走 N 手，到达后自动停止
```

## 4. 人机对战

推荐命令：

```powershell
.\wzq-s1-ASCII.exe --mode 1 --log .\game_log.txt
```

你执黑棋，AI 执白棋。

轮到你时，程序会提示：

```text
Player1 BLACK input coordinate (A1-O15, example H8; type last to show last move):
```

输入格式是：

```text
字母列 + 数字行
```

例如想下在 H 列第 8 行，就输入：

```text
H8
```

列是 `A` 到 `O`，行是 `1` 到 `15`。小写也可以，例如 `h8`。程序也兼容旧写法 `8 8`，会按“第 8 行第 8 列”转换成 `H8`。

如果你想执白棋，让 AI 执黑先走：

```powershell
.\wzq-s1-ASCII.exe --mode 3 --log .\human_white_log.txt
```

模式 3 中，AI 会先下黑棋，然后轮到你输入白棋坐标。

## 5. AI 自博弈

让 AI 自己和自己下：

```powershell
.\wzq-s1-ASCII.exe --mode 2 --log .\ai_vs_ai_log.txt
```

如果想让终端保留每一步棋盘，不要清屏：

```powershell
.\wzq-s1-ASCII.exe --mode 2 --no-clear --log .\ai_vs_ai_log.txt
```

如果只想观察前 50 手：

```powershell
.\wzq-s1-ASCII.exe --mode 2 --no-clear --max-moves 50 --log .\ai_vs_ai_50.txt
```

## 6. 棋盘符号

```text
x    黑棋
o    白棋
X    上一手黑棋，或胜利线上的黑棋
O    上一手白棋，或胜利线上的白棋
+    黑棋当前提示位置
*    白棋当前提示位置
.    空位
```

`Player1` 是黑棋，`Player2` 是白棋。

棋盘上方和下方是字母列 `A-O`，左右两边是数字行 `15-1`。例如 `H8` 就是中间星位附近。

如果想查看上一手，直接看棋盘上大写的 `X` 或 `O`；棋盘下方也会显示：

```text
Last move: Player1 BLACK at H8 by AI.
```

轮到你输入时，也可以输入：

```text
last
```

程序会重新提示上一手坐标。

游戏结束时，程序会额外输出胜利线路，例如：

```text
Player1 wins!
Winning line: D11 E12 F13 G14 H15
```

这样就能直接看出是哪 5 个点连成一线。

## 7. 查看历史日志

如果运行时用了：

```powershell
--log .\game_log.txt
```

那么历史记录会保存在：

```text
D:\cpp\wuziqi\game_log.txt
```

在终端查看：

```powershell
Get-Content .\game_log.txt
```

只看开头：

```powershell
Get-Content .\game_log.txt -TotalCount 80
```

用记事本打开：

```powershell
notepad .\game_log.txt
```

日志里会记录：

```text
Move 001: Player1 BLACK coord=H8 source=AI
```

意思是：第 1 手，黑棋，下在 `H8`，来源是 AI。

## 8. 当前禁手规则

程序只对黑棋检查禁手。白棋没有禁手。

已处理的规则：

```text
黑棋正好五连：黑棋胜，禁手失效
黑棋长连：禁手
黑棋四四：禁手
黑棋三三：禁手
组合禁手：通过四四、三三、长连组合判断
```

注意：复杂三三禁手的判断会递归检查“下一手能否合法形成活四”。这比简单判断慢，但更接近《中国五子棋竞赛规则》的描述。

## 9. 重新训练 AI 权重

训练文件是：

```text
train.py
```

查看训练参数：

```powershell
python .\train.py -h
```

本次训练使用了两段式训练。

第一段，快速粗训练：

```powershell
python .\train.py --generations 60 --population 6 --games 2 --seed 20260511 --out .\runs\fresh_train --forbidden-depth 0 --max-steps 100 --random-rate 0.02
```

第二段，开启三三禁手精修：

```powershell
python .\train.py --generations 72 --population 3 --games 1 --seed 20260511 --out .\runs\fresh_train --forbidden-depth 1 --max-steps 60 --random-rate 0.01 --resume
```

训练结果保存在：

```text
D:\cpp\wuziqi\runs\fresh_train\weights.txt
D:\cpp\wuziqi\runs\fresh_train\training_log.json
```

## 10. 训练参数解释

```text
--generations 60
```

目标训练到第 60 代。配合 `--resume` 时表示“接着训练到第 60 代”，不是“再训练 60 代”。

```text
--population 6
```

每一代尝试 6 组候选权重。越大搜索越充分，也越慢。

```text
--games 2
```

每组候选权重和当前最佳权重对战 2 轮。程序会正反手各下一局，所以实际是 `2 × 2 = 4` 局。

```text
--seed 20260511
```

随机种子。相同种子和参数会更容易复现结果。

```text
--out .\runs\fresh_train
```

训练输出目录。

```text
--resume
```

从已有 `training_log.json` 继续训练。程序每代都会保存，所以中途按 `Ctrl+C` 停止后可以继续。

```text
--random-rate 0.01
```

AI 随机落子的概率。越小越稳定，越大探索越多。

```text
--forbidden-depth 1
```

训练时检查复杂三三禁手的递归深度。越大越接近复杂规则，也越慢。

```text
--max-steps 60
```

训练时每局最多走 60 手，避免训练局拖太久。



## 11. 把训练结果写入 C 程序

训练完成后查看权重：

```powershell
Get-Content .\runs\fresh_train\weights.txt
```

会看到类似：

```c
int weights[FEATURE_COUNT] = {
    138381, -66933, 6693, 2685, 2240, 89, 23, 54497, 6029, 6
};
```

把这组数字填到 `wzq-s1-ASCII.c` 里的 `weights` 数组，然后重新编译：

```powershell
& "D:\Program Files\mingw64\bin\gcc.exe" .\wzq-s1-ASCII.c -o .\wzq-s1-ASCII.exe
```

本次我已经替你填入并编译过。

## 12. 安全训练建议

建议先用小参数测试：

```powershell
python .\train.py --generations 3 --population 3 --games 1 --seed 1 --out .\runs\test --forbidden-depth 0 --max-steps 80
```

确认能跑通后再加大参数。

不建议一开始就把这些参数同时调得很大：

```text
--generations
--population
--games
--forbidden-depth
--max-steps
```

它们都会增加运行时间。训练是单进程 CPU 计算，不会直接损伤硬件；如果电脑变得很热或风扇很响，可以按 `Ctrl+C` 停止。由于程序每代都会保存，下次加 `--resume` 可以继续。

## 13. 常见问题

如果出现：

```text
Set-Location: Cannot find path 'train.py'
```

说明你用了 `cd train.py`。正确做法是进入文件夹：

```powershell
cd D:\cpp\wuziqi
```

然后运行：

```powershell
python .\train.py
```

如果运行 C 程序时日志没有出现，确认命令里写了：

```powershell
--log .\game_log.txt
```

如果没有写 `--log`，程序默认写到：

```text
game_log.txt
```

如果训练中途停了，继续训练示例：

```powershell
python .\train.py --generations 100 --population 6 --games 2 --seed 20260511 --out .\runs\fresh_train --forbidden-depth 1 --max-steps 60 --random-rate 0.01 --resume
```

如果想重新开始训练，换一个新的输出目录，例如：

```powershell
--out .\runs\new_train
```
